Assignment 2 Problem 4. [20 Marks for correctness.]

Write the C implementation date.c for the interface date.h.

Public test samples:

const struct date date1 = {12,33,2013};
const struct date date2 = {12,23,2013};
const struct date date3 = {2,10,2010};
const struct date date4 = {12,23,2013};
const struct date date5 = {2,29,2100};

assert(is_valid(date1)==false);
assert(d_eq(date4 ,date2)== true);
assert(is_valid(date5)==false);
assert(greaterthan(date2,date3)== true);

Notes:

You may assume that the dates will be after January 1, 1900.

3 rules must be taken into account to identify leap years (years where February has 29 days):

  • If the year is evenly divisible by 4 then it is a leap year.
  • If the year is evenly divisible by 100, it is NOT a leap year. This takes precedence over the previous rule.
  • If the year is evenly divisible by 400, it is a leap year. This takes precendece over the previous rule.
